THE RECORD OF CHANGES · BEGINS 2026-07-19

What has changed, and by how much.

Every time a layer is rebuilt, the Ledger compares the figures it is about to publish against the ones already published, and records what moved. This page is that record. It tells you that a figure changed and by how much — it does not tell you why, because the Ledger cannot honestly say.

Three things to know before you read it

  1. The record begins the day it shipped — 2026-07-19. It cannot describe changes made before it existed, because the old figures were overwritten rather than compared. One historical exception is included below, and it is labelled; it is the only one whose cause is known rather than guessed.
  2. It reports that a figure moved, never why. A figure can move because the source restated its own published data, because the source redefined what the figure means, because we corrected an extraction error, or because we changed method deliberately. A restatement and a redefinition are indistinguishable from the outside — they differ only in the source's intent, which lives in its release notes, not in its numbers. The Ledger will not guess between them.
  3. On the State Controller's layers, the earlier figure cannot be reproduced by anyone — including us. Those pipelines ask the Controller's portal to aggregate the rows and return the totals, so the underlying rows never reach this machine. Once the portal revises, the prior figure exists only in this record and in the file it replaced.

Check a citation

Every citation this site produces carries the date its data was generated — data generated 2026-07-14, or Generated 2026-07-18. Paste a citation (or just the date) and this will report whether anything in that layer has changed since.

Reads only the generated date. Nothing is sent anywhere.

How the comparison is made

Figures are compared by identity, not by name — a district's CDS code, a community-college district's MIS code, an agency's identifier and a department's code. A entity that is renamed at the source is not a changed figure, and a rebuild cannot silently reassign a record from one entity to another.

Three kinds of event are recorded, all equally: a figure that changed, a figure that appeared, and a figure that disappeared. The last two matter as much as the first. When the FY 2016-17 city classifier was fixed, money moved between category keys that did not previously exist; counting only changed values would have reported 31 events out of a 482-city correction and missed most of its size.

Comparison is on figures, not file digests. A file's SHA-256 changes when its build date changes, so two builds on the same day can have different digests and identical figures. The record carries a second, figures-only digest — the same hash with the whole metadata block removed — so "did anything actually move" is answerable without trusting a date.

Each batch also carries the pipeline commit it was built from. When that is unchanged between two builds, one possible cause — that we changed the method — is excluded mechanically, without anyone being asked to assert it. Where a source publishes its own "rows last updated" stamp, that is recorded too. It is a source signal, not an explanation.
